# Webデザイン品質指針（LP・単一ページ向け）

## 最重要原則：方向性を1つ決めて振り切る
- 最初に「このページの性格」を1語で決める（例：高級感／親しみ／ミニマル／レトロ／和モダン／ポップ）
- すべての選択（書体・配色・余白・角丸・言葉づかい）をその性格に従わせる
- **禁止：「AIが作った風」の没個性デザイン**＝紫系グラデ背景＋中央寄せカード羅列＋絵文字見出し＋汎用的なうたい文句。これらは性格が要求しない限り使わない

## タイポグラフィ（品質の8割はここで決まる）
- Google Fonts を2書体まで：見出し用（個性が出るdisplay系）＋本文用（可読性重視）
- 日本語の組み合わせ例：
  - 高級・和：見出し「Shippori Mincho」＋本文「Noto Sans JP」
  - 親しみ：見出し「Zen Maru Gothic」＋本文「Noto Sans JP」
  - モダン・テック：見出し「Zen Kaku Gothic New」＋本文「Noto Sans JP」
- 見出しは大きく（hero h1: clamp(2.2rem, 6vw, 4rem)）、字間 letter-spacing を調整
- 本文は 16px 以上・line-height 1.8 前後

## 配色
- ベース（背景）／インク（文字）／アクセント（CTA・強調）の3役を決める。アクセントは1色に絞る
- 性格に合うパレットを選ぶ（高級=深緑×生成り×金、テック=濃紺×白×シアン等）
- 文字と背景のコントラストはWCAG AA（4.5:1）以上

## レイアウト
- 構成の基本：ヒーロー（強いコピー＋CTA）→ 特徴/メニュー → 信頼（実績・声）→ アクセス/FAQ → フッター
- 余白は8pxの倍数システム。セクション間は最低 96px。窮屈にしない
- 1カラム基調＋要所でグリッド（特徴3つ等）。max-width 1100px 前後で中央に
- レスポンシブ必須（メディアクエリ1つ以上、モバイルで崩れない）

## コンテンツ
- **実物のコピーを書く**。「ここにテキストが入ります」「Lorem ipsum」は禁止
- 発注内容から店名・サービス名・特徴を具体的に起こす（不明なら自然な仮名を作る）
- CTAは動詞で具体的に（「予約する」「無料で試す」）

## 実装ルール（単一ファイル）
- 完全な単一 index.html：CSSは `<style>`、JSが必要なら `<script>` に内蔵
- 外部依存は Google Fonts の `<link>` のみ許可（https）。**外部画像URL・プレースホルダ画像サービスは禁止**
  - ビジュアルはCSS（グラデーション・図形・パターン）とインラインSVGで作る
- セマンティックHTML（header/main/section/footer、h1は1つ）＋ `<meta name="viewport">` ＋ `<title>` ＋ lang="ja"
- ページ内リンク（href="#..."）は対応する id を必ず用意する
- hover・スクロールなどの動きは控えめに（transition 0.2〜0.3s）。性格に合う場合のみ

## 出力形式
- html フィールドには生のHTMLのみ（Markdownのコードフェンスで囲まない）
