# Python 実装の一般規約（タスク横断で再利用）

特定フレームワークに依存しない、Pythonコード生成の共通ルール。

## 出力形式（最重要）
- コードはMarkdownのコードフェンス（```）で囲まず、**生のPythonソースだけ**を該当フィールドに入れる。
- そのままファイルに保存して実行できる完全なコードにする（断片や「...省略...」を残さない）。

## コード品質
- 型ヒントを付ける（引数・戻り値）。
- 命名は snake_case（関数・変数）、PascalCase（クラス）。
- 標準ライブラリ + 明示した依存のみ使用。勝手に重い依存を増やさない。
- 例外処理は意味のある単位で。握りつぶさない。
- マジックナンバー・重複は避け、小さく分割する。

## 依存の最小化
- MVPでは外部DB・外部サービスを使わず、インメモリ（dict/list）で完結させる。
- 追加依存を使う場合は、それが本当に必要か考える。
