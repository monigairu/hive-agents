# 単一ファイルWebアプリの実装定石

「ブラウザで開くだけで動く」を絶対条件とする。サーバー・ビルド・外部APIを前提にしない。

## 構成（品質の土台）
- 完全な単一 index.html：CSSは `<style>`、JSは `<script>` に内蔵。外部依存は Google Fonts の `<link>` のみ
- JSは `"use strict"` ＋即時実行関数（IIFE）でグローバル汚染を避ける
- **状態は1つの state オブジェクトに集約**し、変更のたびに render() で画面を描き直す
  （DOMを直接ちまちま書き換える方式はバグの温床。データ→描画の一方向にする）
- ルール判定・計算などのロジックは**純粋関数に分離**する（デバッグ・修正が容易になる）

## localStorage 永続化（ユーザーのデータを扱うアプリは必須）
- キー名はアプリ固有＋バージョン（例: `"kakeibo.v1"`）
- 保存: `localStorage.setItem(KEY, JSON.stringify(state))`。state が変わるたびに呼ぶ
- 復元: `try { JSON.parse } catch { 初期値 }`（壊れたデータで白画面にしない）
- 「データを全部消す」を用意する場合は確認ステップを挟む

## ゲームの定石
- 盤面・手札はデータ（配列）で持つ。DOMをデータ源にしない
- 合法手判定・裏返し・勝敗判定は純粋関数に分離し、境界（盤の端）を丁寧に扱う
- 手番・スコア・終了状態を常に画面に表示。打てない時はパス処理、終局では勝敗を明示
- CPU対戦をつけるなら単純でよい（合法手からランダム／最多獲得）。強さより「ルールが正しい」こと

## 操作性・モバイル
- click イベント基準（タッチでも発火する）。dblclick・hover 依存の操作は禁止
- タップ対象は44px以上。320px幅でも崩れない（グリッドは可変・横スクロールを出さない）
- 破壊的操作（削除・リセット）は誤操作しにくくする（確認 or 取り消し）

## エラーに強く
- **開いた瞬間にJSエラーを出さない**（構文エラー・未定義参照はブラウザ実行検証で機械的に差し戻される）
- ユーザー入力は必ず検証する（数値欄に文字・空入力・負の値）。エラーは画面内に日本語で表示（alert は使わない）
- 空データ状態（初回起動）でも意味のある画面にする（「まだ記録がありません」＋最初の操作への誘導）

## 出力形式
- html フィールドには生のHTMLのみ（Markdownのコードフェンスで囲まない）
